# CLAUDE.md - NeuronXcompta V3

## Project Overview

NeuronXcompta V3 is a full-stack accounting assistant for a dental practice. Migrated from Streamlit (V2) to React + FastAPI. All 21 pages are fully implemented with zero placeholders. Includes a **`ManualAssociationDrawer`** (1100px, `components/justificatifs/ManualAssociationDrawer.tsx`) — 2-column parallel view (ops | justificatifs) with left-slide preview panel, free date/amount filters (±days/€), "Élargir" toggle to bypass backend ±1 month pre-filter, targeted + flux modes, keyboard ⏎/←/→/Esc. Available from JustificatifsPage (header + floating batch bar) and EditorPage (header + selection-triggered bar). Includes a **`JustifToOpDrawer`** (1000px, `components/justificatifs/JustifToOpDrawer.tsx`) — reverse-direction drawer (justif → op) opened from `GedDocumentDrawer` for pending justificatifs, with full-size `PreviewSubDrawer` on the left (700px), inline OCR editing (date/amount/supplier re-runs suggestions), lock indicator + unlock button on candidate rows, self-healing desynced `operation_ref` indices in backend enrichment. Includes a sandbox watchdog that auto-processes files (PDF/JPG/PNG) dropped into `data/justificatifs/sandbox/` with OCR and real-time SSE notifications. Includes a GED (Gestion Électronique de Documents) module for document library with accounting post deductibility tracking. Includes a Dotations aux Amortissements module with registre des immobilisations, moteur de calcul linéaire/dégressif, et détection automatique des opérations candidates. Includes a Tasks Kanban module for tracking accounting actions (auto-generated + manual tasks) with drag & drop. Includes an Export Comptable V3 module with calendar grid, per-month ZIP generation (PDF+CSV+relevés+rapports+justificatifs), export history with ZIP contents expander, and multi-export selection. Includes an Email Comptable module with universal drawer for sending documents to accountant via SMTP Gmail (HTML emails with logo, single ZIP attachment, document selection by type/period, email history with coverage tracking). Includes a **Snapshots** module for named reusable operation selections (ad-hoc folders for follow-ups like "Litige Amazon", "À vérifier comptable"), stored in `data/snapshots.json` with self-healing refs via op-identity hash across file moves (split/merge/archive). Includes **bulk-lock / bulk-unlock** toggle (Verrouiller/Déverrouiller en masse via BulkLockBar) dans JustificatifsPage + EditorPage. Includes a **Note de frais** feature: `source: Optional[str]` field on `Operation` (`"note_de_frais" | "blanchissage" | "amortissement" | None`), amber pill badge `#FAEEDA`/`#854F0B` displayed above Catégorie cell in EditorPage + JustificatifsPage + AlertesPage, split button `+ Ligne ▾` in EditorPage header (main click = op bancaire, chevron = dropdown with `Note de frais (CB perso)`), filter dropdown `Type d'opération` in 4 pages (Éditeur Filtres panel, Justificatifs toolbar, Rapports pill buttons, Compta Analytique `RepartitionParTypeCard` widget with share%). Includes an **on-demand empty month file creation** feature: endpoint `POST /api/operations/create-empty` + service `operation_service.create_empty_file(year, month)` writes `operations_manual_YYYYMM_<hex8>.json`, EditorPage month dropdown shows all 12 months (missing ones displayed as `{Mois} — vide · créer` with value `__CREATE_N__` → triggers `window.confirm` + mutation → auto-select new file). Resolves the UX blocker where logging notes de frais required the bank statement to be already imported. Includes a **MLRetrainToast** (`frontend/src/components/shared/MLRetrainToast.tsx`): animated brain icon with 2 pulse rings, amber + violet pills (corrections count + days since training), persistent top-right toast triggered once per session from `AppLayout` when the auto-task `ml_retrain` exists in the current year's tasks. Includes a **6th auto-detection** in `task_service.generate_auto_tasks`: `ml_retrain` triggers when corrections accumulated since last training exceed configurable thresholds (`ml_retrain_corrections_threshold=10`, `ml_retrain_days_threshold=14` in Settings).

## Architecture

- **Backend**: FastAPI (Python 3.9+), runs on port 8000
- **Frontend**: React 19 + Vite + TypeScript + TailwindCSS 4, runs on port 5173
- **Data**: JSON file storage in `data/` directory. Imports split: `data/imports/operations/` (JSON) and `data/imports/releves/` (PDF). Auto-migration at startup.
- **ML**: Rules-based + scikit-learn categorization, pickle models in `data/ml/`
- **OCR**: EasyOCR with pdf2image, cache `.ocr.json` alongside PDFs. OCR page is the primary entry point for justificatif uploads (batch upload PDF/JPG/PNG + immediate OCR). **Auto-rename post-OCR (filename-first)** : après extraction, les justificatifs sont renommés automatiquement en `fournisseur_YYYYMMDD_montant.XX.pdf` via `rename_service.compute_canonical_name()` qui priorise le parsing du filename existant (3 regex tolérantes) avant de retomber sur l'OCR. Cette stratégie évite les catastrophes quand l'OCR se trompe sur le supplier (mention légale) ou le montant. Helpers : `naming_service.build_convention_filename`, `naming_service.normalize_supplier`, `naming_service.deduplicate_filename`. Renommage met à jour PDF + .ocr.json + associations opérations + GED metadata. **Convention point décimal** (ex. `amazon_20250214_24.00.pdf`). Idempotent. **Endpoint scan on-demand** `POST /api/justificatifs/scan-rename?apply=true&apply_ocr=true` pour rattraper les orphelins ; bouton « Scanner & Renommer » dans OCR > Gestion OCR (drawer avec preview + confirm). **FilenameEditor** : composant frontend inline-editable dans l'historique OCR et le drawer justificatif, avec suggestion convention (bouton Sparkles) et badge "auto" si renommé automatiquement. **Onglet « Gestion OCR »** (renommé depuis « Historique ») : centre de gestion des fichiers OCR avec tri par date de scan (`processed_at`) ou date facture (`best_date`), filtre association (tous/sans/avec), recherche multifocale, bouton crayon par ligne → `OcrEditDrawer` (édition supplier/date/montant/cat/sous-cat + **sélecteur op refondu** en dropdown custom riche avec filtre par mois via chip bleu + recherche textuelle + items date/libellé/montant/badge cat/crédit vert + état vide avec bouton élargir + **nom canonique affiché en live** sous les champs avec 3 états emerald/gris+badge/ambre + rename & associate), bouton orange « Scanner & Renommer » → `ScanRenameDrawer`. `OcrEditDrawer` est aussi accessible **depuis `GedDocumentDrawer`** (badge ambre header + bouton Actions pour les justificatifs).
- **Hints comptables dans `.ocr.json`** : 2 clés top-level `category_hint` + `sous_categorie_hint` stockées dans chaque `.ocr.json` (hors `extracted_data` pour ne pas polluer les arrays OCR). Écrites automatiquement par `justificatif_service.associate()` à chaque association manuelle ou auto (copie `op.Catégorie` / `op.Sous-catégorie`, skip `Autres`/`Ventilé`). Lues par `rapprochement_service.score_categorie()` comme **override prioritaire** de la prédiction ML — un hint présent donne un score catégorie fiable (1.0 si match op, 0.6 sous-cat ≠, 0.0 sinon) au lieu de dépendre de la prédiction ML depuis le supplier parsé. Éditables aussi via `OcrEditDrawer` / `SkippedItemEditor`. Effet en cascade : chaque association enrichit le `.ocr.json`, ce qui booste les futurs rapprochements automatiques sur ce même fichier (même si dissocié plus tard). `_load_ocr_data()` injecte les hints dans le dict `ocr_data` passé à `compute_score()` via `justificatif_ocr.category_hint` / `justificatif_ocr.sous_categorie_hint`. `PATCH /api/ocr/{filename}/extracted-data` accepte aussi ces champs (modèle `OcrManualEdit` étendu).
- **Sandbox Watchdog**: `watchdog` library monitors `data/justificatifs/sandbox/` for PDF/JPG/PNG, auto-converts images to PDF, auto-OCR + auto-rename + SSE notifications (enrichies `auto_renamed` + `original_filename`). `SandboxArrivalToast` affiche le nom original du fichier déposé (`originalFilename`) en titre et le nom canonique post-rename en sous-titre.
- **Image Support**: JPG/JPEG/PNG justificatifs are converted to PDF at intake via Pillow (`_convert_image_to_pdf()`). Only PDF is stored. Constants in `config.py`: `ALLOWED_JUSTIFICATIF_EXTENSIONS`, `IMAGE_EXTENSIONS`, `MAGIC_BYTES`.
- **Amortissements**: Registre des immobilisations avec moteur de calcul **linéaire uniquement** (dégressif interdit en BNC régime recettes — `_compute_tableau_degressif` conservé en lecture des immos legacy). Détection des opérations candidates **stricte `Catégorie == "Matériel"`** (`detect_candidates`, le champ `categories_immobilisables` du config est supprimé). Plafonds véhicules CO2 conservés (`plafond_fiscal`, `co2_classe`). Gestion cessions/sorties (`calculer_cession`, drawer dédié). Champs `Immobilisation` renommés depuis l'ancien schéma : `libelle→designation`, `valeur_origine→base_amortissable`, `duree_amortissement→duree`, `methode→mode`, `poste_comptable→poste`, `quote_part_pro` désormais `float`. Données dans `data/amortissements/` (migration JSON idempotente au lifespan via `_migrate_amortissement_config`). Moteur de calcul dupliqué Python/TypeScript (frontend A2 à mettre à jour pour les nouveaux noms).
- **Reprise d'exercice antérieur (backfill)**: 3 champs optionnels sur `Immobilisation` pour les immos acquises avant utilisation de NeuronXcompta : `exercice_entree_neuronx: int | None` (année à partir de laquelle NeuronX gère l'immo, doit être > année acquisition), `amortissements_anterieurs: float = 0.0` (cumul dotations passées par l'ancien comptable, **non déductible** côté NeuronX), `vnc_ouverture: float | None` (VNC au début de `exercice_entree_neuronx`). Validation Pydantic : `exercice_entree_neuronx > année(date_acquisition)`, `vnc_ouverture >= 0`, cohérence `amortissements_anterieurs + vnc_ouverture == base_amortissable` (tolérance 1 €). Les ops `perso` (catégorie case-insensitive `"perso"`) sont **hors assiette** — ni dans les recettes, ni dans les charges, elles ne réduisent PAS le BNC. `recettes_pro = CA liasse SCP` si la liasse de l'année est saisie (`data/liasse_scp/liasse_{year}.json`), sinon `recettes_pro = sum(crédits bancaires pro)` (proxy provisoire). `charges_pro = sum(débits bancaires pro hors EXCLUDED_FROM_CHARGES_PRO) - sum(csg_non_deductible)`. **Source of truth backend** : `bnc_service.compute_bnc(year)` (façade) qui appelle `analytics_service._bnc_metrics_from_operations()` (lui-même délègue le tri pro/perso/attente + explosion ventilations à `export_service._prepare_export_operations()`) puis ajoute dotations annuelles (`amortissement_service.get_dotations(year).total_deductible`) et forfaits (`charges_forfaitaires_service.get_total_deductible_year(year)`). Imports locaux dans chaque fonction pour éviter cycles. Consommateurs branchés : `analytics_service.get_dashboard_data()` (paramètre `year_full` pour enrichir BNC en année complète), `analytics_service.get_year_overview()`, `fiscal_service.get_historical_bnc()` (annual via bnc_service ; mensuel reste en proxy bancaire — limitation A1, OD mensuelles à venir en Prompt B), `export_service.get_bnc_summary(year)` (façade). Chaque entrée de `category_summary` porte un champ `nature: "pro" | "perso" | "attente"` calculé par `_nature_of_category()`. `get_monthly_trends` retourne 3 séries parallèles `{trends_all, trends_pro, trends_perso}`.

## PDF Import Pipeline

- **Extraction**: `operation_service.extract_operations_from_pdf()` uses pdfplumber + regex
- **Dates**: Extracted as `DD/MM/YY` or `DD/MM/YYYY`, converted to `YYYY-MM-DD` (required by HTML `input type="date"`)
- **Amounts**: Simple pattern `\d+[,.]\d{2}` tried first (avoids false thousands grouping), last match used (amount is rightmost on line)
- **Exclusion filter**: Lines containing SOLDE, TOTAL, etc. are skipped (balance lines, not operations)
- **Categorization at import**: Basic keyword matching (`_categorize_simple()`) at PDF import — priority order matters (e.g. "Remplaçant" with REMPLA keywords checked before "Revenus" with VIREMENT). Full ML categorization (rules + sklearn) runs **automatically** when a file is loaded in EditorPage (empty categories only). ML model supports substring matching in keywords (e.g. "motifremplacementdr" matches keyword "rempla") and `subcategory_patterns` for pattern-based sous-catégorie prediction. Manual "Recatégoriser IA" button available to force re-categorize all lines.
- **Deduplication**: SHA-256 hash of PDF content, first 8 chars in filenames

## Critical Constraints

- **Python 3.9**: MUST use `from __future__ import annotations` in all backend files. Use `Optional[X]` not `X | None`, use `list[X]` only with future annotations.
- **NaN values**: Operation JSON files may contain NaN floats. The `_sanitize_value()` function in `operation_service.py` handles this.
- **PageHeader**: Uses `actions` prop (not children) for header buttons.
- **Dark theme**: All colors via CSS variables in `index.css` (`bg-background`, `bg-surface`, `text-text`, `text-text-muted`, `border-border`).

## Patterns to Follow

- **Hooks**: Use TanStack Query (`useQuery`, `useMutation`, `useQueryClient`) for all API calls
- **Styling**: Tailwind classes only, use `cn()` for conditional classes
- **Icons**: Lucide React (already installed)
- **Drawers**: Fixed panel with `translateX` transition + backdrop, 600-800px wide. GedDocumentDrawer is **resizable** (mousedown drag on left edge, min 400px, max 1200px)
- **Forms**: Controlled components with `useState`, mutations with `onSuccess` invalidation
- **Backend services**: Always call `ensure_directories()` at start, use `from __future__ import annotations`
- **SSE**: Use `StreamingResponse` with `text/event-stream`, send initial `data: {"status":"connected"}` to flush the connection
- **Toasts**: Use `react-hot-toast` (`toast.success()`, `toast.error()`) — `<Toaster />` is in `App.tsx`
- **Sidebar sections**: Utilise `NAV_SECTIONS` avec labels de section discrets (uppercase, text-[10px])
- **File selectors**: EditorPage et AlertesPage utilisent un sélecteur en cascade **année → mois** (pas de dropdown unique surchargé). Fichiers triés chronologiquement. `availableYears` et `monthsForYear` via `useMemo`.
- **Year-wide view**: EditorPage propose "Toute l'année" qui charge tous les fichiers en parallèle via `useYearOperations` (hook `useQueries`). Mode **lecture seule** (pas de save/edit). Badge ambre "Lecture seule — Année complète". Filtres et tri fonctionnels.
- **Category/subcategory filters**: Panel Filtres de EditorPage propose un dropdown catégorie + un dropdown sous-catégorie dépendant (peuplé via `subcategoriesMap`). Reset auto de la sous-catégorie au changement de catégorie. Grille 5 colonnes.
- **Comparatif recettes/dépenses**: ComparatifSection sépare les catégories en 2 groupes (recettes si credit > debit, dépenses sinon). 2 graphiques côte à côte avec légendes dynamiques (périodes sélectionnées). 2 tableaux avec colonnes adaptées (Crédit A/B pour recettes, Débit A/B pour dépenses). Delta badges inversés pour les revenus (hausse = vert). Clic catégorie → CategoryDetailDrawer.
- **Auto-categorization**: EditorPage déclenche automatiquement `categorizeMutation` (mode `empty_only`) au chargement d'un fichier via `useEffect` + `useRef` anti-boucle.
- **EditableCell (EditorPage)** : utilise un état local (`useState`) pour éviter la perte de focus à chaque frappe. La valeur est committée au parent au `onBlur` ou `Enter` (pas à chaque `onChange`). Sync depuis le parent via `useRef` pour undo/categorize.
- **Justificatifs vue opérations** : `useJustificatifsPage` hook dédié. Enrichit les opérations avec `_originalIndex` + `_filename` pour transmettre les index corrects au drawer (pas les index filtrés). Mode "Toute l'année" supporte `_sourceFile`. Filtre par défaut "Sans justificatif". Sync année avec données disponibles (fallback première année si store contient une année absente). **Multi-sélection** : `selectedOps: Set<string>` (clés `filename:index`), `toggleOp`, `toggleAllFiltered`, `clearSelection` — auto-clear quand filtre/année/mois change.
- **Batch reconstitution fac-similé** : `BatchReconstituerDrawer` (550px). Flow : sélection ops dans le tableau → barre flottante "Reconstituer (N)" → drawer avec groupes par template suggéré (match catégorie prioritaire, fallback alias fournisseur). Dropdown template modifiable par groupe. Ops sans template match affichées en warning ambre. Backend : `POST /api/templates/batch-suggest` groupe les ops par meilleur template (`_suggest_by_category()` puis `suggest_template()`), puis `POST /api/templates/batch-generate` génère séquentiellement. Rectangle blanc du fac-similé élargi dynamiquement (`max(largeur_coords, largeur_texte + padding)`) pour couvrir l'ancien montant.
- **PDF preview inline** : endpoint `/preview` utilise `FileResponse` avec `content_disposition_type="inline"` (pas `attachment`). Drawer utilise `<object type="application/pdf">` avec fallback lien. Basename extrait de `Lien justificatif` via `.split('/').pop()` car le champ peut contenir un chemin avec dossier (`traites/...`).
- **PDF thumbnails en vues liste/grille/hover** : règle stricte — **jamais** de `<object type="application/pdf">` ni `<iframe src=".../preview">` dans les grilles, tableaux ou popovers hover. Le plugin PDF du navigateur se décharge silencieusement au-delà de ~5-10 instances et provoque un hard refresh. Utiliser le composant `PdfThumbnail` (`components/shared/PdfThumbnail.tsx`) qui pointe vers les endpoints PNG `/api/justificatifs/{name}/thumbnail` ou `/api/ged/documents/{id}/thumbnail` (cache backend `data/ged/thumbnails/{md5}.png`, 200px de large, régénération si PDF source plus récent). Les drawers de détail (instance unique, visible à la fois) peuvent conserver `<object>` sans problème — c'est uniquement la multiplicité qui casse le plugin PDF. Fix historique : OCR Gestion OCR hover popover 300×400 qui se déchargeait après 30+ hovers → migré vers `<PdfThumbnail justificatifFilename={filename} lazy={false}>`.
- **Image upload**: Justificatifs acceptent PDF/JPG/PNG. Images converties en PDF à l'intake via `_convert_image_to_pdf()` (Pillow). Validation magic bytes multi-format.
- **GED V2 tree**: `build_tree()` returns `{"by_period": [...], "by_category": [...], "by_vendor": [...], "by_type": [...], "by_year": [...]}`. Frontend GedTreePanel with 5 tab icons. Node IDs encode filters (`period-{y}-T{q}`, `cat-{name}`, `vendor-{slug}-{y}`, `releve-{y}-{m}`, `year-{y}-{type}-{m}`). `deriveFiltersFromNode()` in GedTreePanel.tsx converts nodeId → GedFilters. - **Auto-rapprochement v3 — scope mois + cache ML + filename_amount + garde-fou** : `run_auto_rapprochement(year=None, month=None)` accepte des params optionnels qui activent un scope ±1 mois (PDFs et ops filtrés via `_compute_scope_window` + `_extract_pdf_date`). Sans params : scan global. **Pipeline interne** (`_run_auto_rapprochement_locked`) :
  1. `_ml_predict_for_supplier.cache_clear()` au début (intra-run, évite stale après retraining ML).
  2. Skip des PDFs déjà référencés via `justificatif_service.get_all_referenced_justificatifs()` (cache TTL 5s).
  3. Pré-filtre scope-mois sur les PDFs si activé : skip si la date inférée (filename canonique > OCR `best_date`) est hors fenêtre `[mois-1, mois+1]`. Sans date inférable → skip aussi (pas couvert par le scope).
  4. Pré-filtre scope-mois sur les ops dans la boucle interne (cheap, conservé même hors scope pour cohérence).
  5. Pré-filtre date ±15j par PDF (existant, plus tight que la fenêtre scope mais s'applique aussi en mode global).
  6. Calcul du score via `compute_score`. **`compute_score` enrichi** : essaie tous les montants candidats `[filename_amount, best_amount, ...amounts]` pour `score_montant`, prend le max. Robustesse anti-bug OCR (cas openai_24.00.pdf avec `best_amount=824` mais `amounts` contenant 24 — le filename canonique prévaut). `_extract_filename_amount` regex sur `_(\d+(?:\.\d{1,2}))(?:_[a-z0-9]+)*\.pdf$`.
  7. **`auto_total = 0.55*M + 0.45*D`** suit le tracking `best_score` / `second_best_score` (au lieu du `result["total"]` 4-critères).
  8. Gate date existant (rejet si `score_date == 0.0`, écart >14j).
  9. **Garde-fou fournisseur strict** : rejet auto si `best_match.score.detail.fournisseur == 0.0` — sans signal lexical du fournisseur, pas de preuve que ce PDF correspond à cette op (montant + date similaires ne suffisent pas, ex. EDF 48,20€ le 01/05 vs LW MONTAUBAN 47,80€ le 29/04). Suggestions visibles dans le drawer manuel pour validation humaine.
  10. Auto-association si `auto_total >= 0.80` ET delta vs second_best > 0.02. Auto-lock si `auto_total >= 0.95`.
